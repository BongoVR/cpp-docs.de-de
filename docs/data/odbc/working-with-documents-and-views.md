# Arbeiten mit Dokumenten und Ansichten
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Viele Features der MFC\-Bibliothek \(Microsoft Foundation Classes\) stützen sich auf eine Dokumenten\- bzw. Ansichtarchitektur.  Normalerweise werden die Daten in einem Dokument gespeichert. Eine Ansicht zeigt sie im Innenbereich eines Rahmenfensters an und verwaltet die Interaktion des Benutzers mit den Daten.  Die Ansicht kommuniziert mit dem Dokument, um die Daten abzurufen und zu aktualisieren.  Sie können die Datenbankklassen mit oder ohne Framework verwenden.  
  
 Weitere Informationen über die Verwendung der Datenbankklassen im Framework finden Sie unter [MFC: Verwenden von Datenbankklassen mit Dokumenten und Ansichten](../../data/mfc-using-database-classes-with-documents-and-views.md).  
  
 Standardmäßig erstellt der MFC\-Anwendungs\-Assistent ein Anwendungsskelett ohne Datenbankunterstützung.  Sie können jedoch Optionen auswählen, um eine minimale Datenbankunterstützung oder weitergehende formularbasierte Unterstützung zu aktivieren.  Weitere Informationen über Optionen von Anwendungs\-Assistenten finden Sie unter [Datenbankunterstützung, MFC\-Anwendungs\-Assistent](../../mfc/reference/database-support-mfc-application-wizard.md).  
  
 Sie können die Datenbankklassen auch verwenden, ohne von der vollständigen Dokument\-\/Ansichtarchitektur Gebrauch zu machen.  Weitere Informationen finden Sie unter [MFC: Verwenden von Datenbankklassen ohne Dokumente und Ansichten](../../data/mfc-using-database-classes-without-documents-and-views.md).
