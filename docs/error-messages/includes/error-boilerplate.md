Der Visual Studio-Compiler und die Buildtools können viele Arten von Fehlern und Warnungen gemeldet. Nachdem eine Fehlermeldung oder Warnung gefunden wird, möglicherweise Build-Tools Annahmen über Code Absicht und um den Vorgang fortzusetzen, versucht wird, sodass mehrere Probleme gleichzeitig gemeldet werden können. Wenn die Tools allerdings von falschen Annahmen ausgehen, sind spätere Fehler oder Warnungen möglicherweise nicht auf das Projekt anwendbar. Wenn Sie Probleme im Projekt beheben, sollten Sie immer mit dem ersten gemeldeten Problem (Fehler oder Warnung) beginnen, und häufig einen Rebuild für das Projekt ausführen. Eine Lösung nehmen möglicherweise viele nachfolgende Fehler behoben.

Um Hilfe zu einem bestimmten diagnosemeldung in Visual Studio zu erhalten, wählen Sie sie in der **Ausgabe** Fenster, und drücken Sie die **F1** Taste, um die Seite "Dokumentation" für diesen Fehler zu öffnen, falls vorhanden. Sie können auch mit der obigen Suchfunktion der Artikel über bestimmte Fehler oder Warnungen finden, oder Durchsuchen Sie die Liste von Fehlern und Warnungen vom Tool und geben Sie im Navigationsbereich.

> [!NOTE]
> Nicht alle Visual Studio-Fehler oder eine Warnung ist dokumentiert. In vielen Fällen enthält die diagnosemeldung alle Informationen, die verfügbar ist. Wenn Sie auf dieser Seite abgerufen, wenn Sie verwendet **F1** und Sie glauben, dass des Fehlers oder Warnung zusätzlichen Erklärung Bedarf, informieren Sie uns. Sie können den Feedback-Formular auf dieser Seite, fügen Sie im Abschnitt "Kommentare" einen Kommentar hinzu oder [löst ein Problem auf GitHub](https://github.com/MicrosoftDocs/cpp-docs/issues). Sie können auch Feedback senden, und geben Sie Fehler in der IDE. In Visual Studio, wechseln Sie auf der Menüleiste, und wählen Sie **Hilfe > Feedback senden > ein Problem melden**, oder übermitteln Sie einen Vorschlag mit **Hilfe > Feedback senden > Senden Sie einen Vorschlag**.

Sie können zusätzliche Hilfe für Fehler und Warnungen finden Sie in Microsofts öffentlichen Foren. Sie können für den Fehler oder die Warnungsnummer suchen, auf die [Entwicklercommunity](https://go.microsoft.com/fwlink/p/?linkid=820594) Standort. Die [Visual C++-Sprache](http://go.microsoft.com/fwlink/p/?linkid=158195) Forum für Fragen und Diskussionen über die C++-Sprachsyntax und der Compiler MSVC ist. Die [Visual C++ Allgemein](http://go.microsoft.com/fwlink/p/?linkid=158194) -Forum eignet sich für Fragen zu Visual C++, die in anderen Foren nicht diskutiert werden. Kann auch für Fehler und Warnungen zu suchen und Ihre Fragen [Stack Overflow](http://stackoverflow.com/) Lösungen gefunden.
