# Hinzuf&#252;gen von ATL-Unterst&#252;tzung zu einem MFC-Projekt
[!INCLUDE[vs2017banner](../../assembler/inline/includes/vs2017banner.md)]

Wenn Sie bereits eine MFC\-basierte Anwendung erstellt haben, können Sie leicht Unterstützung für Active Template Library \(ATL\) hinzufügen, indem Sie den Assistenten zum Hinzufügen von ATL\-Unterstützung zu einem MFC\-Projekt ausführen.  
  
> [!NOTE]
>  ATL und MFC ist im Allgemeinen nicht in die Express\-Editionen von Visual Studio unterstützt.  
  
> [!NOTE]
>  Die Unterstützung wirkt sich nur auf einfache COM\-Objekte aus, die einer MFC\-Anwendung oder einem DLL\-Projekt hinzugefügt wurden.  Zwar können Sie MFC\-Projekten andere COM\-Objekte \(einschließlich ActiveX\-Steuerelemente\) hinzufügen, aber die Objekte funktionieren möglicherweise nicht erwartungsgemäß.  
  
### So fügen Sie dem MFC\-Projekt ATL\-Unterstützung hinzu  
  
1.  Klicken Sie im Projektmappen\-Explorer mit der rechten Maustaste auf das Projekt, dem Sie ATL\-Unterstützung hinzufügen möchten.  
  
2.  Klicken Sie im Kontextmenü auf **Hinzufügen** und dann auf **Klasse hinzufügen**.  
  
3.  Klicken Sie auf das Symbol **ATL\-Unterstützung zu MFC hinzufügen**.  
  
    > [!NOTE]
    >  Dieses Symbol befindet sich im ATL\-Ordner im Bereich **Kategorien**.  
  
4.  Sobald Sie aufgefordert werden, klicken Sie auf **Ja**, um die ATL\-Unterstützung hinzuzufügen.  
  
 Weitere Informationen darüber, wie der Code des MFC\-Projekts durch integrierte ATL\-Unterstützung geändert wird, finden Sie unter [Details zur ATL\-Unterstützung, die vom ATL\-Assistenten hinzugefügt wird](../../mfc/reference/details-of-atl-support-added-by-the-atl-wizard.md)
