# CA0063
Fehler beim Laden der Regelsatzdatei oder einer ihrer abhängigen Regelsatzdateien.  
  
 Fehler beim Laden der Regelsatzdatei oder einer ihrer abhängigen Regelsatzdateien.  
  
 Der angegebene Regelsatz oder eine der im Regelsatz enthaltenen Regelsatzdateien konnte nicht gefunden werden.  Stellen Sie sicher, dass alle im Regelsatz enthaltenen Regelsätze auf dem Datenträger vorhanden sind und dass die entsprechenden Regelsatzverzeichnisse im Projekt in der **CodeAnalysisRuleSetDirectories**\-Eigenschaft von [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)] angegeben sind.  
  
 Um den Fehler zu debuggen, untersuchen Sie die Regelsatzdatei in einem Text\-Editor.  Um den Pfad zur Regelsatzdatei zu ermitteln, klicken Sie mit der rechten Maustaste im Projektmappen\-Explorer auf das Projekt, klicken Sie auf **Eigenschaften** und anschließend auf **Codeanalyse**.  Stellen Sie sicher, dass die Regelsatzdatei in **Diesen Regelsatz ausführen** ausgewählt ist.  Der Pfad zum Regelsatz wird im Beschreibungsfeld aufgeführt.  
  
 Untersuchen Sie den **Path**\-Attributwert aller **Include**\-Elemente.  Includepfade können relative Pfade verwenden, um die übergeordnete\/aktuelle Regelsatzdatei, Umgebungsvariablen oder absolute Pfade anzugeben.  Beispiel:  
  
```  
<Include Path="%PUBLIC%\Documents\RuleSets\alldesign.ruleset" Action="Default" />  
<Include Path="..\alldesign.ruleset" Action="Default" />  
<Include Path="C:\Rulesets\alldesign.ruleset" Action="Default" />  
```  
  
 Überprüfen Sie die Pfade, und stellen Sie sicher, dass alle Includepfade korrekt sind.  
  
 In einigen Fällen kann die Position eines Regelsatzes von einer [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)]\-Eigenschaft abhängig sein.  [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)]\-Eigenschaften können nicht aus einen Regelsatz verwiesen werden.  Um dieses Problem zu umgehen, geben Sie zusätzliche Suchpfaden in der Eigenschaft **CodeAnalysisRuleSetDirectories** von [!INCLUDE[vstecmsbuild](../build/includes/vstecmsbuild_md.md)].  Geben Sie in diesem Szenario im **Path**\-Attribut des **Include**\-Elements nur den Namen des Regelsatzes an.
